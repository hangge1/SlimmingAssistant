# 开发日志

更新日期：2026-06-26

## 项目起点

项目从 `doc/requirement.md` 出发，目标是做一个个人使用的瘦身助手 Web 应用。第一版定位为自己使用，不做多人账号体系，不接自动健康数据源，只支持手动录入。

## 对话中的需求变更记录

1. 使用范围
   - 第一版只给自己用。
   - 先做 Web，默认跨平台访问。
   - 不做传统账号体系，使用访问设备识别。

2. 访问保护
   - 首次访问设备需要输入一个访问密码。
   - 已信任设备后不再重复输入密码。
   - 密码暂时保存在服务器本地数据库中，只保存哈希。
   - 后续补充了访问密码修改和受信设备移除能力。

3. 数据录入
   - 只做手动录入。
   - 健康记录每天只记录一次，多次提交覆盖当天记录。
   - 跑步记录可以同一天多条。
   - 历史页需要支持补充历史记录，选择日期后进入对应日期的录入页。

4. 目标与反馈
   - 需要设置健康目标和跑步目标。
   - 跑步目标包括每周次数、每周跑量。
   - 健康数据和运动数据同等重要。
   - 首页需要反馈距离目标还有多远、按当前趋势还需多久、给出克制的鼓励文案。
   - BMI 可以显示，身高放到配置界面维护。

5. 配置中心
   - 参数需要可配置。
   - 身高、提醒邮箱、趋势估算下限、提醒规则、SMTP、访问保护均放入配置中心。
   - 趋势估算参数需要系统下限，不能随意设得过低。

6. 提醒
   - 接受邮件提醒和站内提醒。
   - 后续补充 SMTP 配置、测试邮件、邮件提醒失败状态保留。

7. UI 与语言
   - 网站必须是中文。
   - 风格要简洁。
   - 首页不要把“今日状态”放在最显眼的大卡片位置，改为右上角紧凑状态面板。
   - 宽屏下不要留下大面积右侧空白，主内容应使用可用宽度。

8. 数据可视化
   - 首页不能全是文字，需要方便观察的曲线图。
   - 健康曲线和运动曲线不能密密麻麻堆满多张小图。
   - 每类曲线改为单张主图，通过下拉框切换指标。
   - 曲线需要纵轴。
   - 鼠标悬浮到某一天需要浮现该日数值。

9. 跑步配速
   - 平均配速不应由用户手动填写。
   - 配速应由跑步时长 / 公里数自动计算。
   - 录入页和编辑页中的配速改为只读计算项。

10. 开发辅助
    - 原 MCP 浏览器能力在当前会话不可用。
    - 增加本地 Playwright 截图脚本，用真实浏览器渲染桌面、宽屏、移动端截图，辅助后续 UI 调整。

## 2026-06-26 首页与记录体验增量调整

### 调整背景

本轮调整来自对现有页面的实际使用反馈：首页目标反馈不够突出，曲线时间范围固定且切换反馈不明显，记录页更像导航页而不是提交页，历史记录操作按钮位置影响浏览效率。

### 已完成调整

- 首页目标进度前置到首页摘要和数据曲线之前，并放大目标卡片和环形完成度图。
- 目标摘要不再只显示“已设置”，改为直接显示健康目标或跑步目标的剩余差距，并在描述中展示当前值、目标值和跑步目标差距。
- 健康曲线和运动曲线增加时间范围下拉，支持最近 7 天、最近 1 个月、最近半年、最近 1 年，默认最近 7 天。
- 修复健康曲线切换到最近 1 个月时看起来没有变化的问题：图表横轴现在使用所选时间范围的固定起止日期，而不是仅用实际数据点的起止日期。
- 移除底部重复的“趋势和 BMI”区块；趋势统一通过数据曲线查看，当前 BMI 放入健康摘要，BMI 趋势保留在健康曲线指标中。
- 记录页改为“提交记录”工作台：顶部选择记录日期，下面直接展示健康记录和跑步记录两个提交表单。
- 历史记录卡片的编辑、删除操作移动到每条记录标题行右侧，去掉底部独立操作栏。
