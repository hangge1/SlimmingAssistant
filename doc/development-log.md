# 开发日志

更新日期：2026-06-26

## 项目起点

项目从 `doc/requirement.md` 出发，目标是做一个个人使用的瘦身助手 Web 应用。第一版定位为自己使用，不做多人账号体系，不接自动健康数据源，只支持手动录入。

## 对话中的需求变更记录

1. 使用范围
   - 第一版只给自己用。
   - 先做 Web，默认跨平台访问。
   - 不做传统账号体系，使用访问设备识别。

2. 访问保护
   - 首次访问设备需要输入一个访问密码。
   - 已信任设备后不再重复输入密码。
   - 密码暂时保存在服务器本地数据库中，只保存哈希。
   - 后续补充了访问密码修改和受信设备移除能力。

3. 数据录入
   - 只做手动录入。
   - 健康记录每天只记录一次，多次提交覆盖当天记录。
   - 跑步记录可以同一天多条。
   - 历史页需要支持补充历史记录，选择日期后进入对应日期的录入页。

4. 目标与反馈
   - 需要设置健康目标和跑步目标。
   - 跑步目标包括每周次数、每周跑量。
   - 健康数据和运动数据同等重要。
   - 首页需要反馈距离目标还有多远、按当前趋势还需多久、给出克制的鼓励文案。
   - BMI 可以显示，身高放到配置界面维护。

5. 配置中心
   - 参数需要可配置。
   - 身高、提醒邮箱、趋势估算下限、提醒规则、SMTP、访问保护均放入配置中心。
   - 趋势估算参数需要系统下限，不能随意设得过低。

6. 提醒
   - 接受邮件提醒和站内提醒。
   - 后续补充 SMTP 配置、测试邮件、邮件提醒失败状态保留。

7. UI 与语言
   - 网站必须是中文。
   - 风格要简洁。
   - 首页不要把“今日状态”放在最显眼的大卡片位置，改为右上角紧凑状态面板。
   - 宽屏下不要留下大面积右侧空白，主内容应使用可用宽度。

8. 数据可视化
   - 首页不能全是文字，需要方便观察的曲线图。
   - 健康曲线和运动曲线不能密密麻麻堆满多张小图。
   - 每类曲线改为单张主图，通过下拉框切换指标。
   - 曲线需要纵轴。
   - 鼠标悬浮到某一天需要浮现该日数值。

9. 跑步配速
   - 平均配速不应由用户手动填写。
   - 配速应由跑步时长 / 公里数自动计算。
   - 录入页和编辑页中的配速改为只读计算项。

10. 开发辅助
    - 原 MCP 浏览器能力在当前会话不可用。
    - 增加本地 Playwright 截图脚本，用真实浏览器渲染桌面、宽屏、移动端截图，辅助后续 UI 调整。

## 2026-06-26 首页与记录体验增量调整

### 调整背景

本轮调整来自对现有页面的实际使用反馈：首页目标反馈不够突出，曲线时间范围固定且切换反馈不明显，记录页更像导航页而不是提交页，历史记录操作按钮位置影响浏览效率。

### 已完成调整

- 首页目标进度前置到首页摘要和数据曲线之前，并放大目标卡片和环形完成度图。
- 目标摘要不再只显示“已设置”，改为直接显示健康目标或跑步目标的剩余差距，并在描述中展示当前值、目标值和跑步目标差距。
- 健康曲线和运动曲线增加时间范围下拉，支持最近 7 天、最近 1 个月、最近半年、最近 1 年，默认最近 7 天。
- 修复健康曲线切换到最近 1 个月时看起来没有变化的问题：图表横轴现在使用所选时间范围的固定起止日期，而不是仅用实际数据点的起止日期。
- 移除底部重复的“趋势和 BMI”区块；趋势统一通过数据曲线查看，当前 BMI 放入健康摘要，BMI 趋势保留在健康曲线指标中。
- 记录页改为“提交记录”工作台：顶部选择记录日期，下面直接展示健康记录和跑步记录两个提交表单。
- 历史记录卡片的编辑、删除操作移动到每条记录标题行右侧，去掉底部独立操作栏。

## 2026-07-08 账户、邮件提醒、自动部署和首页体验固化

### 调整背景

本轮调整来自云端部署和真实使用反馈：需要最简账号隔离、管理员/普通用户/访客模式、普通用户收件邮箱、邮件提醒可靠触发、宝塔自动化部署、登录页和首页视觉动效，以及首页目标卡片结构重新梳理。

### 已完成调整

- 账号体系改为轻量实现：管理员可在设置中新增、修改、删除普通用户；普通用户只访问自己的个人数据；访客模式只做临时体验，不持久化数据。
- 默认管理员账号由环境或初始化逻辑确定，早期测试环境可使用 `admin / admin`，生产部署后应尽快修改密码，避免浏览器和密码管理器提示弱口令或泄露风险。
- 管理员创建用户时填写的显示名称作为该用户默认昵称；普通用户进入设置页时应显示这个昵称，并允许自行修改。
- SMTP 发信服务器由管理员统一维护；普通用户只设置自己的提醒收件邮箱，并提供测试邮件按钮。
- 邮件提醒后台任务由生产启动脚本拉起，每分钟检查提醒规则；提醒事件按提醒时间区分，用户当天调整提醒时间或收件邮箱后允许重新触发，不再用“同一用户同一天已有任意邮件事件”全局阻断。
- 每日提醒时间改为小时和分钟两个下拉框，分钟支持 `0-59`，保存时仍转换为 `HH:mm`。
- 登录页和首次创建管理员页增加全屏运动背景、鼠标移动反馈和点击反馈；登录后的应用背景铺满整个视口，并尊重 `prefers-reduced-motion`。
- 首页目标区由“目标体重 / 目标腰围”两张卡片调整为“健康目标 / 运动目标”两张卡片：健康目标承载体重和腰围，运动目标承载每周跑步次数和每周跑量。
- 首页状态配色固化：未设置目标使用蓝灰中性提示；健康目标使用翠绿色系；运动目标使用运动蓝色系；今日打卡未完成使用红色，完成使用翠绿色。
- 自动化云端部署脚本落地到项目内，发布包上传到 `/www/wwwroot`，依赖安装、数据库迁移、`better-sqlite3` alias 修复、旧发布包清理和宝塔 Node 项目修复由脚本处理；服务器凭据不写入仓库，通过环境变量或执行时补齐。

### 防回归约定

- 后续 UI 调整不得把首页第一屏目标区改回单独的“目标体重 / 目标腰围”卡片。

## 2026-07-09 打卡输入范围与交互优化

### 调整背景

打卡页原先存在两个问题：健康数据使用 0.1 粒度下拉框，体重这类字段会生成上千个选项；跑步数据使用自由文本输入，心率、步频、步幅等指标缺少常规范围约束，容易录入脏数据。

### 已完成调整

- 健康打卡和健康记录编辑改为数值输入，保留单位栏，并显示常规人体范围提示。
- 跑步打卡和跑步记录编辑改为数值输入，距离、时长、心率、步幅、步频分别设置合理的 `min` / `max` / `step`。
- 时长、平均心率、步频要求整数；步幅支持 0.01 米；体重、围度、体脂率和跑步距离支持小数步进。
- 配速继续只读自动计算，并在后端拒绝明显不合理的公里数和时长组合。
- 前端表单和后端解析共用 `features/records/constants/record-input-rules.ts`，避免 UI 范围和服务端校验分叉。

### 防回归约定

- 不要把体重、腰围、臀围、体脂率改回 0.1 粒度的超长下拉框。
- 不要把跑步距离、时长、心率、步幅、步频改回自由文本输入。
- 修改打卡字段范围时必须同步更新共享规则、后端测试和 UI 截图。

## 2026-07-15 个人站首页与项目页视觉重设计

- 统一个人站卡片、按钮、阴影、留白、移动端断点和 hover 反馈。

## 2026-07-17 复位产品定位、自我定位 3D 秩序地图与社区方向迭代

### 调整背景

本轮迭代来自对“复位”网站定位的重新收敛：产品不应被写成心理疾病诊断或治疗工具，而应帮助处在失序、焦虑、内耗、行动断裂中的用户，用更低负担的方式识别当前状态，并进入可执行、可记录、可复盘的自我修复路径。

产品结构重新收敛为两块：

- 自我定位：帮助用户看清当前主要失序来源，识别心理秩序、认知秩序、行动秩序哪一条更需要优先修复。
- 自我修复：承接定位结果，进入心理重建、认知重塑、行为重构三条修复方向。

同时明确“复位社区”的定位：展示用户明确公开后的复位历程，帮助其他用户看到“别人也卡过、也能一点点复位”，而不是做排行榜、成功学展示或打卡广场。

### 需求精华

- 用户往往说不清为什么失序，也没有足够认知完成长问卷，因此界面要优先提供选择题，并保留可选自定义输入。
- 页面文案必须像面对面对话一样说话，避免生硬商业话术。
- 自我定位要像一轮一轮的状态对话，而不是一次性表单。
- 不使用“诊断”“治疗”等医疗化定位，优先使用“自我定位”“复位报告”“修复方向”“复盘再校准”等表达。
- 自我定位的交互形式要降低焦虑用户的完成压力，因此从问卷式体验升级为 3D 轻游戏式秩序地图。

### 已完成调整

- 新增并更新产品硬约束文档：`doc/product-constraints.md`。
- 新增本轮聚合记录：`doc/resetlife-product-positioning-2026-07-17.md`。
- `AGENT.md` 改为引用独立产品约束文档，避免把产品规则混在代理执行说明里。
- 全站导航和核心页面围绕“自我定位 / 自我修复 / 复位记录 / 复位社区”重新收敛。
- 首页改为 Reset 品牌入口，强化 3D Reset 按钮、暗场探照和高端品牌表达。
- `/state` 改为 Three.js 3D 秩序地图：
  - 未选择起点前不显示星球。
  - 选择心理秩序、认知秩序或行动秩序后弹出对应星球。
  - 顶部显示当前秩序状态条，并提供箭头式“重选”。
  - 状态碎片沿星球赤道外侧一圈排列，带稳定距离和轻微随机偏移。
  - 左键按住拖动时，星球和碎片使用同一套 3D 坐标系一起旋转。
  - 普通鼠标移动不触发旋转。
  - 鼠标滚轮支持 zoom，星球、碎片轨道、碎片缩放和遮挡圆盘同步缩放。
  - 背面碎片进入星球圆盘范围时，会变暗、模糊并被星球前景遮挡层盖住，形成正面/背面关系。
  - 点击碎片后在碎片附近弹出程度选择卡片。
- `/path` 承接自我定位结果，输出复位报告和优先修复路径。
- `/psychology`、`/cognition`、`/action` 分别承接心理重建、认知重塑、行为重构方向。
- `/records` 保持为用户自己的复位记录。
- `/community` 保持为公开复位历程展示，不混同私有记录。

- 不要把复位社区做成排行榜、打卡广场或成功学展示。
