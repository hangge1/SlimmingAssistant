# 复位产品硬约束

本文档集中维护“复位”网站的产品体验约束。后续新增页面、改文案、改交互时，必须先遵守这里的规则。

## 用户前提

- 复位面向处在失序状态中的用户。
- 用户往往说不清原因，也没有足够多的认知来完成大段自我分析。
- 产品要先帮用户识别状态，再引导用户进入下一步，而不是要求用户一开始就解释清楚自己。

## 交互约束

- 表单交互必须优先让用户做选择题，通过选项帮助用户识别状态、情绪、问题和下一步行动。
- 每组选择题应提供可选的自定义输入，让用户能用自己的话补充或替代选项，避免机械感和被模板定义。
- 状态识别体验必须像一轮一轮的状态对话，而不是一次性大表单。
- 状态识别优先采用卡片式分步对话：每轮只问一个主题，先让用户选择或补充，再进入下一轮。
- 状态识别卡片要突出陪伴感和疗愈感：语气像面对面对话，节奏要慢，避免一次性把很多问题压给用户。

## 文案约束

- 页面文案必须像面对面对话一样直接对用户说话。
- 优先使用“你”和“我们”。
- 避免生硬的商业话术、营销口号和站在外部描述用户的表达。
- 避免医疗化表达。不要把产品写成心理疾病诊断、治疗或复诊；优先使用“状态识别”“状态对话”“复位路径”“修复方向”“复盘再校准”。
- 不要把复位写成内容库、课程销售页或励志宣传页。
- 复位首先是陪用户从混乱中做出一个小选择、完成一个小动作的操作界面。

## 复位社区约束

- “我的复位记录”和“复位社区”必须区分清楚：前者是用户自己的私有记录，后者是用户明确选择公开后的历程展示。
- 复位社区展示的是“复位历程”，不是成功学展示、排行榜或打卡广场。
- 用户记录默认不公开；任何公开行为都必须由用户明确选择。
- 公开内容应优先匿名或弱身份展示，避免暴露敏感隐私细节。
- 社区内容重点展示“我当时卡在哪里 -> 我做了什么小动作 -> 后来有什么变化”。
- 社区互动应鼓励共鸣、收藏和“我也经历过”，避免点赞攀比、排名刺激和制造焦虑。
